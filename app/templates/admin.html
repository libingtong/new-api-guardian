规则配置

规则分为两类：错误事件规则和稳定性规则，分别处理实时故障和反复抖动的渠道

错误事件规则用于“禁用渠道 / 禁用模型”；稳定性规则用于“多次自动禁用后转为人工恢复模式”。
风险提醒：不要创建“全空条件”或范围过宽的规则。未限定渠道、分组、模型的规则可能误伤大量渠道。

错误事件规则

基于错误日志内容命中后执行动作，适合处理接口报错、上游无返回、特定状态码等问题

ID 规则 状态 动作 阈值 匹配摘要 操作

稳定性规则

当某个渠道在一定时间内反复被自动禁用时，将其转入更严格的人工恢复模式

ID 规则 状态 动作 阈值 适用范围 操作

新增规则

按渠道、模型、状态码、错误码、返回内容组合定义独立规则

保存前提醒：如果没有限定渠道、分组、模型，规则覆盖面会非常大。系统已禁止“全空条件”规则。
